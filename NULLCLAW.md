# NullClaw — Agent Runtime Integration

NullClaw is a **678KB Zig binary** with zero external dependencies. It implements a vtable-driven agent loop: receive a user message, route to a configured LLM provider, execute tool calls against registered device handlers, return a response. The Agicore `NULLCLAW` declaration generates the config JSON and the TypeScript tool bridge that connects NullClaw's tool calls to Tauri device commands.

---

## Declaring NullClaw

```agi
NULLCLAW {
  PATH "~/.nullclaw/config.json"
  PROVIDERS {
    ollama "http://localhost:11434"                    1
    babyai "https://novasynchris-babyai.hf.space"    2
  }
  TOOLS {
    read_temperature   sensor_temperature_read
    read_humidity      sensor_humidity_read
    set_led            actuator_status_led_set
    move_servo         actuator_pan_servo_set
    speak              tts_speak
    listen             stt_listen
  }
  PERSONALITY "You are Nova, a helpful home assistant robot. You can read environmental sensors and control lights and servos. Be concise and safe."
}
```

### Fields

| Field | Required | Description |
|-------|----------|-------------|
| `PATH` | No | Path to NullClaw config file (default: `~/.nullclaw/config.json`) |
| `PROVIDERS` | No | Block of `name "url" priority` entries — NullClaw tries providers in priority order |
| `TOOLS` | No | Block of `tool_name maps_to_command` pairs |
| `PERSONALITY` | No | System prompt injected into every NullClaw conversation |

---

## Provider priority

NullClaw routes requests through providers in ascending priority order. Lower number = try first:

```agi
PROVIDERS {
  ollama "http://localhost:11434"                    1   # local Ollama — free, fast, offline
  babyai "https://novasynchris-babyai.hf.space"    2   # BabyAI cloud — smarter, costs tokens
}
```

**Ollama first** is the recommended pattern for embedded devices. Local inference (Qwen 3 0.6B, Phi-3 Mini, Gemma 2B) handles most requests; BabyAI escalates for hard problems or when confidence is low.

If a provider is unavailable (network down, Ollama not running), NullClaw automatically falls to the next priority. This makes the system offline-resilient by default.

## Tool bindings

Each `TOOLS` entry maps a NullClaw tool name (what the LLM invokes) to a Tauri command name (what gets called in your Rust backend):

```agi
TOOLS {
  read_temperature   sensor_temperature_read   # LLM calls "read_temperature" → Tauri calls "sensor_temperature_read"
  set_led            actuator_status_led_set   # LLM calls "set_led" → Tauri calls "actuator_status_led_set"
}
```

The tool name is what the LLM sees in its tool list. The command name must match a `#[tauri::command]` function in your Rust backend — usually generated by SENSOR and ACTUATOR declarations or written in a protected implementation file.

## What gets generated

### `scaffold/nullclaw/config.json`

The NullClaw config file. Copy this to `~/.nullclaw/config.json` on the device, or deploy it as part of your provisioning flow:

```json
{
  "version": "1",
  "personality": "You are Nova, a helpful home assistant robot...",
  "providers": [
    { "name": "ollama", "url": "http://localhost:11434", "priority": 1, "type": "local" },
    { "name": "babyai", "url": "https://novasynchris-babyai.hf.space", "priority": 2, "type": "remote" }
  ],
  "tools": [
    { "name": "read_temperature", "maps_to": "sensor_temperature_read", "transport": "tauri_ipc" },
    { "name": "set_led", "maps_to": "actuator_status_led_set", "transport": "tauri_ipc" }
  ],
  "safety": {
    "max_tokens_per_turn": 2048,
    "require_confirmation": false,
    "log_all_tool_calls": true
  }
}
```

### `src/lib/nullclaw.ts`

TypeScript functions for the Tauri app to start/stop the agent and dispatch tool calls:

```typescript
// Start the NullClaw agent process
await startNullclawAgent();

// Stop it
await stopNullclawAgent();

// Dispatch a tool call manually (NullClaw calls this via IPC)
const result = await dispatchNullclawTool('read_temperature', { unit: 'celsius' });
```

Also exports:
- `NullclawToolName` — union type of all declared tool names
- `dispatchNullclawTool()` — routes a tool name + args to the correct Tauri command

### `src-tauri/src/nullclaw.rs`

Rust commands for launching and managing the NullClaw process:

```rust
// In your protected lib.rs:
mod nullclaw;

.invoke_handler(tauri::generate_handler![
    nullclaw::nullclaw_start_agent,
    nullclaw::nullclaw_stop_agent,
])
```

The generated launcher uses `std::process::Command` to spawn the NullClaw binary. Mark the file `// @agicore-protected` to customize the launch arguments, environment variables, or IPC channel.

## BabyAI integration

BabyAI (`https://novasynchris-babyai.hf.space`) is an OpenAI-compatible inference endpoint that NullClaw can use as a cloud fallback. When declared as a provider, NullClaw will route to it when:

- Ollama is not running or unreachable
- The local model returns low confidence
- The request requires reasoning beyond what the local model can handle

**Cost management:** BabyAI is not free for heavy use. NullClaw's `max_tokens_per_turn` safety field caps per-request token spend. For high-volume devices, prefer local-only routing and escalate to BabyAI only for specific intents.

## Tool design patterns

### Map sensor reads to single-responsibility tools

```agi
TOOLS {
  # Separate tools per sensor — LLM can ask for only what it needs
  read_temperature   sensor_bme280_temperature
  read_humidity      sensor_bme280_humidity
  read_soil          sensor_soil_moisture_read
  read_imu           sensor_imu_read_all
}
```

### Map actuator commands to safe operations

```agi
TOOLS {
  # Named actions, not raw values — prevents LLM from sending out-of-range commands
  set_status_green   actuator_status_led_green
  set_status_red     actuator_status_led_red
  set_status_off     actuator_status_led_off
  speak              tts_speak           # text-to-speech output
  rotate_pan_left    actuator_pan_servo_left
  rotate_pan_right   actuator_pan_servo_right
  rotate_pan_center  actuator_pan_servo_center
}
```

Named tool actions are safer than raw value commands — the LLM cannot send `set_servo(999)` if the only servo tool is `rotate_pan_left`.

### Voice pipeline tools

```agi
TOOLS {
  speak     tts_speak         # Piper TTS → speaker
  listen    stt_listen_once   # Whisper.cpp → text (blocks until spoken)
  alert     alert_play_sound  # plays a sound file, non-blocking
}
```

## Running NullClaw manually

```bash
# Install NullClaw on the device
curl -L https://github.com/novasyn/nullclaw/releases/latest/download/nullclaw-aarch64 -o ~/.local/bin/nullclaw
chmod +x ~/.local/bin/nullclaw

# Copy generated config
cp scaffold/nullclaw/config.json ~/.nullclaw/config.json

# Run interactively
nullclaw --config ~/.nullclaw/config.json

# Run as a service (Tauri app manages this via nullclaw_start_agent command)
nullclaw --config ~/.nullclaw/config.json --daemon
```

## Offline operation

NullClaw is designed offline-first. If all providers fail:

1. It returns a canned response from the personality prompt's fallback rules
2. It logs the failure for next phone-home telemetry upload
3. Rule-based tool calls (wake word, simple commands) still work via the classifier tier

For voice assistants, add a local classifier as a zero-priority provider for intent detection:

```agi
PROVIDERS {
  classifier "file:///opt/novasyn/models/intent.tflite" 0  # instant, offline
  ollama     "http://localhost:11434"                    1  # local LLM
  babyai     "https://novasynchris-babyai.hf.space"    2  # cloud
}
```

## Safety

NullClaw tool calls go through the Tauri command layer, which means your Rust backend can validate every tool invocation before execution:

```rust
// src-tauri/src/commands/actuators.rs  // @agicore-protected
#[tauri::command]
pub async fn actuator_drive_left_set(value: f64) -> Result<(), String> {
    // Clamp to safe range — never trust LLM-supplied values directly
    let clamped = value.clamp(0.0, 1.0);
    // Apply slew rate limit before sending to body
    // ...
    Ok(())
}
```

**Never pass raw LLM tool arguments directly to hardware.** Always clamp, validate, and apply slew limits in the Rust command layer.
