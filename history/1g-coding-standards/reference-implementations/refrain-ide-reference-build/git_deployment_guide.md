# Git & Deployment Automation Guide

_Updated: November 18, 2025_

Refrain IDE 2.0 now automates git workflows and Fly.io deployments as part of the five-step IDE cycle. This guide explains how the backend services are wired, how to configure them, and how to recover from common failures.

---

## 1. Architecture Overview

| Component | Responsibility | Key Files |
|-----------|----------------|-----------|
| `GitService` | Wraps GitPython to expose status, commit, push, diff, and log operations. Used by `/api/v1/git/**` and the workflow engine. | `backend/services/git_service.py` |
| `CommitMessageBuilder` | Generates semantic commit messages that include workflow metadata, AI role, and file stats. | `backend/services/git_service.py` |
| `DeploymentService` | Invokes `flyctl` for backend/frontend deploys, performs health checks, and streams events. | `backend/services/deployment_service.py` |
| Workflow Hooks | After Step 001/004 the workflow auto-stages, commits, pushes, and deploys (configurable). | `backend/services/workflow_service.py` |
| `StrategicChatService` | Drives Step 000 conversations, persists decisions, and feeds workflow checkpoint actions. | `backend/services/strategic_chat_service.py`, `_project_docs/strategic_chat_guide.md` |
| Frontend Git/Deploy Panels | `GitPanel` + `DeploymentPanel` expose git status/actions and Fly deploy orchestration on the dashboard (`frontend/app/page.tsx`). | `frontend/components/GitPanel.tsx`, `frontend/components/DeploymentPanel.tsx` |

## 2. Git Automation

### 2.1 Automatic Commits
- Enabled when `GIT_AUTO_COMMIT=true`.
- After a step completes, the workflow stages files (`git add -A`), analyzes file types, and builds a conventional commit message:
  ```
  feat(api): wire deployment streaming [STEP 004]

  Generated by: backend
  Duration: 42.1s
  Files touched: 6
  ```
- Pushes occur when `GIT_AUTO_PUSH=true` **and** the step ID is in `GIT_PUSH_ON_STEPS` (default: `step_001`, `step_004`).

### 2.2 Manual CLI/API Usage
- `GET /api/v1/git/status` to inspect branch + dirty files.
- `POST /api/v1/git/add` + `POST /api/v1/git/commit` to craft manual commits (body must include `message` or set `auto_message=true`).
- `POST /api/v1/git/push` to push a specific branch.
- `GET /api/v1/git/diff?staged=true` to review the index before committing.

### 2.3 Commit Message Tips
- Use scopes: `ui`, `api`, `docs`, `workflow`, `deploy`, etc.
- Keep subjects ≤ 72 chars.
- Include step markers `[STEP 00X]` so audit logs align with workflow history.

## 3. Deployment Automation

### 3.1 When Deployments Trigger
| Step | Action |
|------|--------|
| Step 001 – Frontend AI | Deploys `frontend/` to the staging Fly app (`FRONTEND_APP_NAME`). |
| Step 004 – Backend AI | Deploys `backend/` to the staging Fly app (`BACKEND_APP_NAME`). |

The workflow streams events back to the UI via SSE (see `/deploy/backend/{environment}` and `/deploy/frontend/{environment}`).

### 3.2 Health Checks
- After deploy, the workflow hits `FRONTEND_HEALTHCHECK_URL` or `BACKEND_HEALTHCHECK_URL`.
- Results surface in the SSE stream (`event: deploy_health`).
- Configure timeouts via `DEPLOY_HEALTH_CHECK_TIMEOUT`.

### 3.3 Manual Operations
- `POST /api/v1/deploy/backend/staging` to force a backend deploy.
- `GET /api/v1/deploy/status/{app}` for Fly app status + version info.
- `POST /api/v1/deploy/rollback/{app}?version=<image>` to roll back.
- `GET /api/v1/deploy/logs/{app}` to stream Fly logs (plain text).

## 4. Configuration Reference

Add the following to `.env` (values shown are defaults):

```env
# Git
GIT_AUTO_COMMIT=true
GIT_AUTO_PUSH=true
GIT_PUSH_ON_STEPS=step_001,step_004
GIT_COMMIT_INCLUDE_METADATA=true
GIT_COMMIT_INCLUDE_FILE_STATS=true

# Deployment
DEPLOY_AUTO_ENABLED=true
DEPLOY_ON_STEPS=step_001,step_004
DEPLOY_ENVIRONMENT=staging
DEPLOY_HEALTH_CHECK_TIMEOUT=60
DEPLOY_RETRY_ON_FAILURE=true
DEPLOY_MAX_RETRIES=2

# Fly.io
FLYCTL_PATH=flyctl
BACKEND_APP_NAME=refrain-ide-api
FRONTEND_APP_NAME=refrain-ide-web
BACKEND_HEALTHCHECK_URL=https://refrain-ide-api.fly.dev/health
FRONTEND_HEALTHCHECK_URL=https://refrain-ide-web.fly.dev/api/health
```

> **Tip:** Lists (e.g., `GIT_PUSH_ON_STEPS`) accept comma-separated strings and are split automatically.

## 5. Troubleshooting

| Symptom | Diagnosis | Resolution |
|---------|-----------|------------|
| `git` endpoints return `GitPython is required` | Dependency missing on server | `pip install GitPython` or install via Docker image rebuild. |
| Auto-commit reports “No changes detected” | Files already committed or untracked files excluded | Toggle `include_untracked` via `/git/add` or ensure generator created changes. |
| Deploy endpoint returns `flyctl CLI is not installed` | `flyctl` missing from PATH | Install Fly CLI and authenticate (`fly auth login`). |
| Health check fails after deploy | App not healthy or URL misconfigured | Confirm `*_HEALTHCHECK_URL` values and inspect `/deploy/logs/{app}` for runtime errors. |
| Stage fails due to large binary | `DocumentService` protects `_project_docs` only; repo files still staged | Update `.gitignore`/workflow to skip large artifacts. |
