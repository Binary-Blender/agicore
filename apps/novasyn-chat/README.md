# NovaSyn Chat 2.0

**The first real application built on Agicore.**

A multi-provider AI chat client with conversation context, folder-based knowledge management, session history, and tag-driven retrieval. Built on Tauri 2 + React + Zustand + SQLite, generated from a single `.agi` source file.

This app exists for two reasons:
- **As a working tool** — the author's daily-driver chat client, replacing the 3G NovaSyn Chat Lite (Electron).
- **As a reference implementation** — showing what an Agicore-generated app looks like in practice. Every recurring bug here became an Agicore framework fix. Every hand-extension here is a candidate for future codegen.

---

## What's generated vs. hand-written

### Generated by Agicore (do not edit — regenerated from `novasyn_chat.agi`)

- `src-tauri/src/ai_service.rs` — multi-provider streaming chat, model discovery, API key storage
- `src-tauri/src/commands/*.rs` — CRUD commands per entity, including filtered `list_by_<parent>` variants when applicable
- `src-tauri/src/db.rs` — SQLite connection, migration runner
- `src-tauri/migrations/001_initial.sql` — schema + idempotent SEED rows
- `src-tauri/Cargo.toml`, `tauri.conf.json`, `capabilities/default.json`, icon placeholders
- `src/lib/types.ts`, `src/lib/api.ts` — TS types and invoke wrappers
- `src/store/appStore.ts` — Zustand store with entity slices, `selectedModel`, `currentSessionId`, and per-CURRENT-entity loaders
- `src/components/ModelPicker.tsx`, `src/components/ApiKeyModal.tsx`

### Hand-written (extends the codegen)

- `src-tauri/src/main.rs` — wires the generated ai_service commands into the Tauri builder
- `src/components/ChatView.tsx`, `Sidebar.tsx`, `MessageInput.tsx`, `MarkdownRenderer.tsx`, `SearchBar.tsx`, `ContextBar.tsx`, `TagPicker.tsx`, `DocumentEditor.tsx`, `ExchangeLibrary.tsx`, `FolderPanel.tsx`, `NavRail.tsx`, `SettingsView.tsx`, `TagManager.tsx`, `TerminalView.tsx`, `TitleBar.tsx` — UI shell

The split shrinks over time as Agicore grows new emitters.

## The DSL source

The entire shape of this app — entities, relationships, AI providers, models, navigation context, session entity declaration, FK ordering, default user seed — is declared in `novasyn_chat.agi` (~440 lines).

Reading that file is the fastest way to understand both Agicore and this app.

## Setup

### Prerequisites
- Node 18+
- Rust toolchain (`rustup` from https://rustup.rs/)
- On Windows: MSVC Build Tools (C++ workload) — needed for any Rust app on Windows
- On Linux: `webkit2gtk-4.1` + `librsvg2-dev` (for Tauri webview rendering)

### Run dev

```bash
cd apps/novasyn-chat
npm install
npm run tauri:dev
```

First run takes 1-3 minutes (Cargo compiles ~300 crates). Subsequent runs are seconds. A native window opens with hot-reload on the React side.

### Build a distributable

```bash
npm run tauri:build
```

Output: `src-tauri/target/release/bundle/msi/*.msi` on Windows, `.app`/`.dmg` on macOS, `.deb`/`.AppImage` on Linux. Plus a bare exe/binary in `target/release/`.

### Regenerate from `.agi`

After modifying `novasyn_chat.agi`, regenerate the codegen files:

```bash
cd ../../core/compiler
npx tsx -e "
import { readFileSync, writeFileSync } from 'fs';
import { compile } from './src/index.ts';
const src = readFileSync('../../apps/novasyn-chat/novasyn_chat.agi', 'utf-8');
const result = compile(src);
for (const [path, content] of result.files) {
  writeFileSync('../../apps/novasyn-chat/' + path, content);
}
console.log('Regenerated', result.files.size, 'files');
"
```

(A `npm run regen` script in this directory is on the to-do list.)

## Configuring API keys

On first run, click the **🔑 API Keys** button in the sidebar. Keys are stored at:
- Windows: `%APPDATA%\NovaSyn\api-keys.json`
- macOS/Linux: `~/.config/NovaSyn/api-keys.json`

The same file is shared with NovaSyn Chat Lite (3G), so keys configured in one app are available to the other.

Supported providers (declared in `AI_SERVICE.PROVIDERS`):
- `anthropic` — Claude (sk-ant-...)
- `openai` — GPT-4o, o-series reasoning (sk-...)
- `google` — Gemini (AIza...)
- `xai` — Grok (xai-...)
- `babyai` — HuggingFace Inference (hf_...) — key-storage only; routed through ROUTER (Phase 3, pending)

## Architecture notes

This app is the test bench for the Agicore framework. As of agicore commit `20a03e8` (May 2026), the codegen pipeline handles:

- AI_SERVICE → Rust streaming for multiple providers
- ENTITY → SQL migration + Rust CRUD + TS types + invoke wrappers + Zustand slice
- ENTITY.ORDER → SQL ordering direction
- ENTITY.SEED → idempotent INSERT OR IGNORE rows
- ENTITY.BELONGS_TO + APP.CURRENT → SQL-pushdown filtered list queries + matching store actions
- AI_SERVICE.MODELS (multi-model with LABEL/DEFAULT) → ModelPicker.tsx
- AI_SERVICE.PROVIDERS → ApiKeyModal.tsx (with key-storage-only support)

Future phases will add:

- ACTION emitter (orchestrator-pattern Rust) — `send_chat`, `council_chat`, `broadcast_chat`, `web_search`
- ROUTER emitter (tiered routing + Mosh Pit + preference scoring)
- Document filesystem emitter (`<document>` stream tag scanner)
- SETTINGS / VAULT emitter
- Workspace runtime DB switching

See [`../../README.md`](../../README.md) for the framework overview, [`../../ROADMAP.md`](../../ROADMAP.md) for the plan, and the commit history of this repo for the framework's evolution as it tracked this app's needs.
